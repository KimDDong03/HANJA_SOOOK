# 08. Selected Screen Specification

## 목적

이 문서는 `WIREFRAME_DECISION_PACK.md`의 `선택안 v1`을 구현 가능한 화면 명세로 고정한다.
구조는 이 문서의 텍스트 명세를 기준으로 하되, 시각 스타일은 선택된 생성형 화면 이미지의 비율, 색감, 카드 위계, 버튼 스타일을 구현 기준으로 삼는다.

## 전체 구조

```txt
Splash
→ Role Select
→ Student Setup
→ App Shell
  → 홈
  → 학습
  → 도전
  → 설정
```

## 공통 원칙

- 학생 앱의 하단 탭은 `홈 / 학습 / 도전 / 설정`이다.
- 기본 학생 흐름은 선택지를 많이 보여주지 않고 한 방향으로 이어진다.
- 홈은 오늘 해야 할 한 가지 미션을 바로 시작하는 화면이다.
- 학습은 현재 한자와 오늘 5자 진행을 확인하고 기본 루틴으로 이어가는 화면이다.
- 도전은 기본 도전을 바로 시작하고, 다른 게임은 보조 선택으로 둔다.
- 설정은 role별 연결/관리/역할 변경 화면이다.
- Growth는 별도 탭으로 두지 않고 홈 요약, 결과 화면, 성장 상세 진입으로 분산한다.
- 학부모/선생님 기능은 학생 홈에 같은 위계로 노출하지 않는다.
- 누락 데이터가 있어도 demo flow는 막지 않는다.
- 시각 스타일은 흰 배경, 은은한 한자 패턴, 얇은 테두리 카드, 붉은 Primary CTA, 역할/학년별 보조 컬러를 기본으로 한다.

## Simple Flow v1

학생 기본 플로우:

```txt
홈
→ 오늘의 한자 시작
→ 직접 써보기
→ 자유쓰기 채점
→ 결과
→ 다음 한자 시작 또는 도전하기
```

원칙:

- 기본 CTA는 화면마다 하나만 둔다.
- 모드 선택 화면은 유지하되 기본 흐름에서는 노출하지 않는다.
- 학습 탭은 단계별 버튼 목록이 아니라 현재 한자, 오늘 5자 진행, 루틴 요약을 보여준다.
- 도전 탭의 Primary CTA는 `혼합 퀴즈`으로 바로 들어간다.
- 판뒤집기, 퀴즈 유형 선택, 반 랭킹은 도전 탭의 보조 선택으로 둔다.

## S-01 Role Select

목적:

- 첫 진입 또는 데모 진입에서 사용자 role을 선택한다.
- 학생을 기본 진입으로 강조하되 학부모/선생님 진입도 제공한다.

구성:

- 앱 로고/이름
- 짧은 태그라인
- 큰 `학생` 카드
- 보조 `학부모` 카드
- 보조 `선생님` 카드
- 각 카드 우측 이동 아이콘

문구:

| 요소 | 문구 |
|---|---|
| 학생 | 한자를 재미있게 배워요 |
| 학부모 | 우리 아이 학습 현황을 확인해요 |
| 선생님 | 학생들의 학습을 관리해요 |

동작:

- `학생` 선택: `Student Setup`으로 이동
- `학부모` 선택: 학부모 role 설정 또는 연결 화면으로 이동
- `선생님` 선택: 선생님 role 설정 또는 Teacher Preview로 이동
- 이미 학생 프로필이 있으면 이후 실행에서는 Home으로 바로 진입 가능

상태:

- Loading: role/session 확인 중
- Empty: 첫 실행 상태
- Error: role 저장 실패 시 재시도 가능

## S-02 Student Setup

목적:

- 학생 이름, 학교/소속, 학년을 입력하고 학생 앱을 시작한다.
- 기존 MVP의 학교 검색 정책을 유지한다.

구성:

- 상단 뒤로가기
- 제목: `학년 선택`
- 학교 검색 입력
- 검색 결과 목록
- 선택된 학교 카드
- 이름 입력
- 학년 선택 그리드
- 시작하기 버튼

학년 정책:

| 학년 | 상태 |
|---|---|
| 3학년 | 활성 |
| 4학년 | 활성 |
| 5학년 | 활성 |
| 6학년 | 활성 |

검증:

| 상황 | 문구 |
|---|---|
| 학교 미선택 | 학교를 선택해주세요. |
| 이름 없음 | 이름을 입력해주세요. |
| 학년 없음 | 학년을 선택해주세요. |
| 검색어 부족 | 학교명을 2글자 이상 입력해주세요. |
| 검색 결과 없음 | 검색 결과가 없습니다. 학교명을 다시 확인해주세요. |
| 네트워크 오류 | 인터넷 연결을 확인한 뒤 다시 시도해주세요. |

CTA:

- Primary: `시작하기`
- Disabled 조건: 학교 미선택, 이름 없음, 학년 미선택

## S-03 Home

목적:

- 오늘 해야 할 학습을 가장 빠르게 시작한다.
- 성장/XP는 요약으로만 보여준다.

구성:

- 학생 이름 기반 인사
- 오늘의 한자/오늘의 미션 카드
- 진행률 `N/4`
- Primary CTA: `오늘의 한자 시작`
- XP/레벨 요약
- 하단 탭: `홈` 활성

이동:

- `오늘의 한자 시작`은 한자 카드 화면을 거치지 않고 `직접 써보기`로 이동한다.

우선순위:

1. 오늘의 미션
2. 성장 요약

상태:

- Loading: 오늘의 학습 불러오는 중
- Empty: 오늘의 한자 데이터가 아직 없습니다.
- Error: 오늘의 한자를 불러오지 못했습니다.

## S-04 Learn

목적:

- 학습 탭의 첫 화면으로, 현재 한자와 오늘 5자 진행을 보여준다.

구성:

- 오늘의 학습 제목
- 오늘의 5자 진행
- 현재 학습 중인 한자 강조
- Primary CTA: `이어서 학습`
- 읽기 전용 루틴 요약
- 하단 탭: `학습` 활성

학습 루트:

```txt
직접 써보기
→ 자유쓰기 채점
→ 결과
```

상태:

- 완료: 체크/별 표시
- 현재: 강조
- 잠김: 이전 단계 필요
- fallback: 획순 데이터 없음이면 자유쓰기 채점으로 이동 가능

## S-05 Hanja Card

목적:

- 한자의 기본 정보를 학습하고 다음 학습 단계로 연결한다.

구성:

- 단원명
- 현재 위치 `N/M`
- 큰 한자
- 음
- 훈/뜻
- 획수
- 예문
- 이미지 또는 fallback 카드
- 다음 단계 CTA

CTA:

- Primary: `쓰기 연습 시작`

누락 처리:

| 누락 | 처리 |
|---|---|
| 이미지 없음 | 기본 카드 배경 표시 |
| 예문 없음 | 예문 영역 숨김 |
| 획순 없음 | 획순 보기 대신 자유쓰기 안내 |

## S-06 Writing Select

목적:

- 학생이 쓰기 모드를 고민하지 않고 바로 연습 또는 채점으로 들어가게 한다.

구성:

- 현재 한자 요약
- 직접 써보기 카드
- 무안내 채점 진입

루틴:

```txt
1. 직접 써보기
2. 자유쓰기 채점
```

모드:

| 모드 | 안내 수준 | CTA |
|---|---|---|
| 직접 써보기 | 전체 회색 글자 가이드 | 시작 |
| 자유쓰기 채점 | 없음 | 채점하기 |

## S-07 Direct Writing

목적:

- 회색 글자 전체 가이드 위에 직접 써보게 한다.
- 획순은 기본 노출하지 않고, 학생이 필요할 때만 버튼으로 확인한다.

구성:

- 현재 한자
- 뜻과 낱말/문장 예시
- 회색 전체 글자 가이드가 깔린 쓰기 캔버스
- `획순 보기` 버튼
- 다시쓰기 버튼
- 성공 피드백 후 자동으로 다음 연습/자유쓰기 채점 이동
- 다음 연습/자유쓰기 채점 버튼은 fallback으로 유지

기본 루틴:

```txt
1. 획순을 보고 따라쓰기
2. 한 번 더 따라쓰기
3. 마지막으로 따라쓰기
4. 자유쓰기 채점
```

오답 피드백:

| 상황 | 처리 |
|---|---|---|
| 위치/비율이 크게 다름 | 해당 위치만 한 번 강조 |
| 획순을 보고 싶음 | `획순 보기` 버튼으로 애니메이션 재생 |
| 연습 완료 | `잘했어요!` 피드백 후 다음 연습으로 자동 이동, 3회 완료 후 `자유쓰기 채점`으로 이동 |

fallback:

- 획순 데이터가 없으면 `자유쓰기 채점`으로 이동 가능
- demo flow를 막지 않는다.

## S-08 Free Writing Score

목적:

- 아무 안내 없이 직접 쓰고 채점받는다.
- Hanja demo의 무안내 채점 흐름도 이 화면 계열로 처리한다.
- 숫자 점수는 학생에게 노출하지 않고, 통과/다시 연습 정도만 결과에서 보여준다.

구성:

- 상단 뒤로가기
- 작은 목표 한자 또는 목표 정보
- 자유쓰기 캔버스
- 채점받기 버튼

루틴:

```txt
자유쓰기
→ 채점받기
→ 결과 화면
```

판정:

| 상황 | 처리 |
|---|---|
| 기준 이상 | 결과 화면에 `자유쓰기 통과` 표시 |
| 기준 미만 | 결과 화면에 `자유쓰기 다시 연습` 표시 |
- 큰 빈 캔버스
- 다시쓰기
- Primary CTA: `채점받기`

채점 후 표시:

- 점수
- 별/랭크
- 간단 피드백
- 다시 도전
- 다음 한자
- 결과 보기

원칙:

- 쓰는 중에는 가이드, 획순, 정답선을 보여주지 않는다.
- 점수 계산은 UI가 아니라 도메인 서비스에서 처리한다.

## S-09 Challenge

목적:

- 도전 탭의 허브로 퀴즈, 게임, 반 경쟁을 진입시킨다.

구성:

- 내 반/내 순위 요약
- 오늘 도전 점수
- Primary CTA: `오늘 도전 시작`
- 퀴즈 카드
- 판뒤집기 카드
- 반 랭킹 카드
- 하단 탭: `도전` 활성

우선순위:

1. 반 경쟁 요약
2. 오늘 도전 CTA: 혼합 퀴즈 바로 시작
3. 퀴즈/게임 모드 보조 선택
4. 랭킹 상세

## S-10 Quiz Modes

목적:

- 퀴즈 종류를 명확히 선택하게 한다.

구성:

- 제목: `퀴즈 선택`
- 모드 카드
- 난이도/예상 XP
- 최근 기록

모드:

| 모드 | 설명 |
|---|---|
| 한자 보고 훈음 | 한자를 보고 음/뜻을 맞힌다. |
| 훈음 보고 한자 | 음/뜻을 보고 한자를 맞힌다. |
| 혼합 퀴즈 | 여러 유형을 섞어서 푼다. |

별도 분리:

- `뜻 보고 한자 선택`은 속도/콤보형 게임으로 도전 탭에 둔다.

## S-11 Quiz Play

목적:

- 공통 퀴즈 프레임에서 문제 유형에 따라 문제/답변 영역만 바꾼다.

공통 구성:

- 진행 `N/10`
- 문제 유형 배지
- 점수/XP
- 문제 카드
- 답변 영역
- 즉시 피드백
- 다음 버튼

유형:

| 유형 | 문제 카드 | 답변 영역 |
|---|---|---|
| 한자 보고 훈음 | 큰 한자 | 음/뜻 선택지 |
| 훈음 보고 한자 | 음/뜻 텍스트 | 한자 선택지 |
| 혼합 퀴즈 | 유형별 변경 | 유형별 변경 |

동작:

- 답 선택 즉시 정답/오답 피드백
- 마지막 문제 후 Result로 이동

## S-12 Flip Board Game

목적:

- 판을 직접 선택하고 답을 맞혀 내 색으로 뒤집는 도전 게임이다.
- 기본형, 리버스형, 반 경쟁형, 라운드 전략형을 모두 채택한다.

공통 규칙:

- 앱이 문제를 하나씩 순서대로 제시하지 않는다.
- 학생이 뒤집고 싶은 판을 직접 선택한다.
- 진입 전에 `훈음 보고 한자 그리기` 또는 `한자 보고 뜻 쓰기` 모드를 선택한다.
- `훈음 보고 한자 그리기` 모드는 판에 훈음만 보여주고, 선택한 판 아래 캔버스에 한자를 그려서 답한다.
- `한자 보고 뜻 쓰기` 모드는 판에 한자만 보여주고, 선택한 판 아래 입력창에 훈음 또는 뜻을 적어서 답한다.
- 정답이면 선택한 판이 내 색으로 바뀐다.
- 제한시간 동안 더 많은 판을 점유하거나 더 높은 점수를 얻는다.

모드:

| 모드 | 판 내용 | 답변 방식 |
|---|---|---|
| 훈음 보고 한자 그리기 | 훈음 | 한자 쓰기 |
| 한자 보고 뜻 쓰기 | 한자 | 훈음 또는 뜻 작성 |
| 반 경쟁형 | 선택한 판뒤집기 모드 | 반 친구 기록과 경쟁 |
| 라운드 전략형 | 난이도/점수 다른 판 | 선택한 판에 답변 |

구성:

- 남은 시간
- 내 점유 판 수
- 반/상대 점유 수
- 판 그리드
- 선택한 판 강조
- 답변 영역
- 정답/오답 피드백
- 콤보/XP

MVP 절충:

- 실시간 대전이 아니라 비동기 반 랭킹으로 시작한다.
- 결과는 Class Ranking에 반영한다.

## S-13 Class Ranking

목적:

- 같은 반 학생끼리 XP, 오늘 도전, 판뒤집기 기록을 비교한다.

구성:

- 반 이름
- 내 순위 카드
- 필터: `XP / 오늘 / 판뒤집기`
- 랭킹 리스트
- 내 기록 강조
- 도전하러 가기 CTA

개인정보 원칙:

- 실명 노출을 최소화한다.
- 닉네임 또는 가림 이름을 사용한다.
- 교사용 상세 정보는 Teacher Preview/교사 role에서만 다룬다.

## S-14 Result

목적:

- 학습/퀴즈/게임 완료 후 성장 보상과 다음 행동을 보여준다.

구성:

- 격려 메시지
- 별/랭크
- XP bar
- 획득 XP
- 활동별 지표
- 다음 행동 CTA

활동별 지표:

| 활동 | 지표 |
|---|---|
| 쓰기 | 정확도, 점수, 별 |
| 퀴즈 | 정답 수, 정답률, 소요시간 |
| 판뒤집기 | 뒤집은 판 수, 점수, 순위 영향 |

CTA:

- 학습 완료: 다음 한자 시작, 도전하기
- 도전 완료: 다시 도전, 랭킹 보기, 도전으로

## S-15 Settings

목적:

- Role Select에서 선택한 role에 따라 다른 설정 항목을 보여준다.
- 학생 학습 흐름과 학부모/교사/연결 기능을 분리한다.

공통:

- 현재 role 표시
- 역할 변경
- 앱 정보
- 로그아웃 또는 초기화

학생 role:

- 내 프로필
- 학교/학년 정보
- 반 참여
- 학습 설정
- 역할 변경

학부모 role:

- 학생 연결 코드 입력
- 연결된 학생 목록
- 학습 현황 진입
- 알림 설정
- 역할 변경

선생님 role:

- Teacher Preview
- 샘플 반/학생 현황
- 반 코드/데모 반 정보
- 역할 변경

원칙:

- Settings는 role별 화면이다.
- 모든 role에 같은 설정 목록을 보여주지 않는다.
- Student Links와 Teacher Preview는 Home이 아니라 Settings에서 다룬다.
