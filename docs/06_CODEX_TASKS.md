# 06. Codex Implementation Tasks

Codex must implement tasks in order.
Do not proceed to the next ticket unless the current ticket passes its acceptance criteria.

## Ticket 00. Flutter Project Bootstrap

### Goal
Create the Flutter project foundation using the architecture in `docs/01_ARCHITECTURE.md`. If `pubspec.yaml` does not exist, initialize the Flutter app in the repository root with `flutter create . --platforms=android,ios` and preserve existing docs/supabase/tools files.

### Create/modify

- `pubspec.yaml`
- `android/` and `ios/` generated by Flutter if absent
- `lib/main.dart`
- `lib/app/app.dart`
- `lib/app/router.dart`
- `lib/app/theme.dart`
- `lib/app/env.dart`
- placeholder screens for all MVP routes

### Dependencies

Use:

- supabase_flutter
- flutter_riverpod
- go_router
- freezed_annotation
- json_annotation
- drift
- drift_flutter
- flutter_svg
- stroke_order_animator

Add dev dependencies for generators/tests.

### Routes

```txt
/
/login
/home
/hanja/:hanjaId
/writing/:hanjaId
/quiz
/game
/result
/growth
/teacher-preview
```

### Acceptance

- App starts at SplashScreen.
- Splash moves to LoginScreen after short delay.
- Login has temporary button to Home.
- Every route renders a placeholder.
- Supabase env structure exists but secrets are not hard-coded.
- `flutter analyze` passes.

## Ticket 04. School CSV Import Script

### Goal
Build a script that converts the provided school CSV into Supabase seed CSV.

### Input

`data/source/학교기본정보_2026년03월31일기준.csv`

### Output

`supabase/seed/schools_elementary_seed.csv`

### Requirements

- Read with `cp949`.
- Filter `학교종류명 == '초등학교'`.
- Map columns exactly as in `docs/05_SCHOOL_DATA_SPEC.md`.
- Trim whitespace.
- Drop rows with empty `행정표준코드` or `학교명`.
- Remove duplicate `standard_school_code`.

### Acceptance

- Script runs from repo root.
- Output CSV exists.
- Output contains only elementary schools.
- Output columns match `schools` import columns.
