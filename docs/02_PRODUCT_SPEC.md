# 02. Product Specification

## 1차 MVP 범위

### 포함

| 화면 | 목적 |
|---|---|
| Splash | 앱 진입 |
| Login | 학교 검색, 이름/학년 입력 |
| Student Home | 오늘의 미션, 학습 진입 |
| Hanja Card | 한자 정보 학습 |
| Writing | 획순/자유 따라쓰기 |
| Quiz | 4지선다 학습 확인 |
| Typing Selection Game | 뜻 보고 한자 선택 게임 |
| Result | 점수, 정답률, XP, 별점 |
| Growth | 레벨, XP, 배지 |
| Teacher Preview | 시연용 교사 화면 |

### 제외

- 실서비스 관리자 웹
- 교사 과제 배포 실제 저장/알림
- 학부모 앱
- AI handwriting scoring
- 음성인식
- 실시간 랭킹
- 결제
- SNS 공유

## Login 정책

### 화면 플로우

```txt
학교명 검색
→ 학교 선택
→ 이름 입력
→ 학년 선택
→ 시작하기
→ anonymous sign-in
→ profile upsert
→ home
```

### 학교 검색

- 소스: Supabase `schools`
- 원본: `data/source/학교기본정보_2026년03월31일기준.csv`
- 표준학교코드: CSV `행정표준코드`
- MVP 대상: `학교종류명 == '초등학교'`
- 검색어 최소 길이: 2자
- 최대 검색 결과: 20개
- 결과 표시: 학교명 / 시도명 / 도로명주소

### 검증 문구

| 상황 | 문구 |
|---|---|
| 학교 미선택 | 학교를 선택해주세요. |
| 이름 없음 | 이름을 입력해주세요. |
| 학년 없음 | 학년을 선택해주세요. |
| 검색어 부족 | 학교명을 2글자 이상 입력해주세요. |
| 검색 결과 없음 | 검색 결과가 없습니다. 학교명을 다시 확인해주세요. |
| 네트워크 오류 | 인터넷 연결을 확인한 뒤 다시 시도해주세요. |

## 학년/오늘의 학습 정책

- 학년 선택: 3, 4, 5, 6
- 기본 daily_hanja_count: 5
- 첫 진입: 선택 학년의 첫 단원/첫 한자부터 표시
- 단원 선택 UI는 MVP에서 숨김
- 한자 데이터 부족 시 demo fallback 표시

## Hanja Card

### 표시 요소

- 단원명
- 한자 대형 표시
- 음
- 훈/뜻
- 획수
- 관련 이미지
- 예문
- 획순 보기
- 따라쓰기
- 퀴즈 시작
- 이전/다음
- 현재 위치 N/M

### Empty 처리

| 누락 | 처리 |
|---|---|
| 이미지 없음 | 기본 카드 배경 표시 |
| 예문 없음 | 예문 영역 숨김 |
| 획순 없음 | 획순 보기 비활성 또는 자유쓰기 안내 |

## Writing

### MVP 원칙

- 시연 흐름을 절대 막지 않는다.
- 획순 데이터가 있으면 guided mode.
- 획순 데이터가 없으면 free mode.
- 결과 점수는 좌표 기반 단순 점수.
- 45점 미만이어도 결과 화면 이동 가능.

### 별점

| 정확도 | 별점 | 메시지 |
|---:|---:|---|
| 85 이상 | 3 | 아주 잘했어요! |
| 65~84 | 2 | 잘했어요! |
| 45~64 | 1 | 조금만 더 연습해요. |
| 45 미만 | 0 | 다시 써볼까요? |

## Quiz

- 문제 수: 10
- 유형: 한자→음, 한자→뜻, 뜻→한자
- 보기: 4지선다
- 정답 선택 즉시 피드백
- 마지막 문제 후 결과 화면 이동

## Typing Selection Game

실제 키보드 타이핑은 MVP에서 구현하지 않는다.

- 방식: 뜻 보고 맞는 한자를 선택
- 보기: 4~6개
- 제한시간: 문제당 20초
- 문제 수: 10
- 점수: 정확도 + 속도 + 콤보

## Teacher Preview

MVP에서는 실제 교사 기능이 아니라 시연용 샘플 화면이다.

포함:

- 학급명
- 학생 수
- 진행 중 과제 요약
- 학생별 완료 여부/정확도/점수 샘플 테이블

제외:

- 과제 생성 실제 저장
- 개별 학생 배포
- 마감일 알림
- 결과 export
